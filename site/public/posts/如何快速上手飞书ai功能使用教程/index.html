如何快速上手飞书AI功能使用教程

May 11, 2026 · 4 min · AI Tools

每天花2小时整理会议纪要、1小时写周报，还要翻遍聊天记录找三个月前的方案——这是大多数团队用飞书前的常态。这份 飞书 AI 功能使用教程 直接带你跳过摸索期：10 分钟内学会用 AI 机器人做知识问答、自动总结群聊精华、甚至用 DALL·E 在群内生成配图。以上这些，读完就能用。

飞书AI功能的入口与准备工作#

飞书 AI 功能的使用入口分两种情况，取决于你是使用飞书官方内置的 AI 能力，还是接入第三方的 AI 智能体（如 LinkAI）。两者入口不同，准备工作的重点也不一样。

找到官方 AI 的入口#

飞书 2025 新版（版本号 ≥ 7.20）已将 AI 能力整合为“飞书智能伙伴”。入口位于桌面端左侧导航栏的机器人图标（一个圆形光点），点击后会弹出 AI 对话窗口。移动端则在聊天列表顶部有一个浮动的 AI 输入条。

如果看不到这个入口，原因通常是：

  • 飞书客户端版本低于 7.20 —— 去官网下载最新版。
  • 企业管理员未在管理后台开启“飞书智能伙伴”功能权限。

注意： 飞书内置 AI 的知识库问答依赖企业管理员预先上传企业文档并授权范围。个人用户无法自行导入资料。如需要私有知识库问答，必须使用第三方智能体方案。

准备第三方 AI 智能体（以 LinkAI 为例）#

接入外部 AI 机器人，准备工作集中在管理员侧。以下是具体步骤：

  1. 在飞书开放平台创建应用：登录 open.feishu.cn，创建企业自建应用，获取 App ID 和 App Secret。
  2. 配置权限：在应用详情页的“权限管理”中，勾选 im:message（消息读写）、im:resource（获取图片与文件）和 drive:drive:readonly（知识库读取，如需要）。
  3. 在 LinkAI 平台完成授权：将上一步的 App ID 和 App Secret 填入 LinkAI 的飞书机器人配置页面。
  4. 发布应用并开启事件：回到飞书开放平台，在“事件与回调”中开启 接收消息 事件，最后点击“版本管理与发布”提交审核。

注意： 飞书自建应用从提交到审核通过通常需要 5-30 分钟。审核期间机器人不可用。

确认基础配置#

无论使用哪种方案，都需要检查以下两项：

  • 企业版本：飞书 AI 功能在“标准版”和“旗舰版”中可用。“免费版”仅能使用基础对话，无法调用 DALL·E 图片生成、网页速读、群聊精华总结等高级插件。
  • 管理员权限：自建应用和插件授权的操作，都需要拥有飞书管理后台的“管理员”或“应用开发者”角色。

完成上述准备后，你就可以在任意群聊中 @AI 机器人或直接私聊它执行具体任务。接下来将介绍如何让 AI 总结群聊消息、生成周报和调用图片生成插件。

使用AI智能问答进行企业知识库查询#

飞书 AI 的知识库查询能力，分为内置智能伙伴和第三方智能体两种实现方式。前者依赖管理员在企业后台导入文档并授权范围；后者（如 LinkAI）允许用户通过私聊或群聊，直接向 AI 提问，AI 会从已关联的飞书知识库中检索答案，并支持返回图片、文件、视频等素材。

使用内置 AI 查询企业知识库#

确保管理员已在飞书管理后台完成以下两项配置：

  • 在“飞书智能伙伴”功能中，上传企业文档（支持飞书云空间内的文档、表格、多维表格）。
  • 在“知识库权限”中，为 AI 功能设置可访问的知识库范围（如按部门、文件夹粒度授权）。

配置完成后，用户在 AI 对话窗口输入问题即可。例如：“上季度Q2的销售目标是多少？”“新员工入职流程中的培训材料在哪里？”AI 会从授权知识库中提取相关片段，并附带原文链接。

提示： 内置 AI 的知识库查询仅返回文本片段，不支持实时读取外部网页或 PDF。如果需要结合外部信息，可以同时 @官方 AI 的“网页速读”插件。

使用第三方 AI 智能体查询知识库（以 LinkAI 为例）#

第三方方案的优势在于支持私有化知识库和自定义语料。以下是在飞书群聊中使用的典型流程：

  1. 私聊或 @机器人：在任意群聊中 @LinkAI 机器人，或在私聊中直接发送消息。
  2. 提问核心要点：问题越具体，答案越准确。如：“帮我在知识库‘市场部’文件夹里找‘2025年品牌策略.pptx’”。
  3. 获取素材：LinkAI 机器人会直接返回文件链接（.pdf/.pptx/.docx）或视频播放地址，无需手动翻文件夹。

常见查询场景包括：

  • 制度查询：“调休申请审批流程是什么？”
  • 项目资料：“A项目最终验收报告在哪里？”
  • 联系人搜索：“找一下技术部张经理的联系方式。”
  • 产品手册：“X产品最新版用户手册有没有更新？”

注意： 第三方智能体需要提前在 LinkAI 平台完成飞书知识库的接入授权（已在准备工作一节说明）。如果未配置，机器人会返回“未找到相关知识”。

提升查询准确率的两个技巧#

  • 问题中带上限定词：如“技术部”而非“我们部门”，“2025年1月”而非“上个月”。
  • 使用“帮我在知识库中查找……”句式：这能明确告诉 AI 搜索范围，避免 AI 调用通用大模型能力而给出猜测性回答。

无论是内置 AI 还是第三方方案，核心逻辑相同：AI 先将你的自然语言问题转化为向量检索，再从知识库中匹配最相关的文档片段，最后用大模型组织成回答。理解这一点，你就能写出更少歧义的问题。

创建专属智能体实现业务自动化#

创建专属智能体的核心，是将飞书聊天界面变成你的业务中枢。本教程（即这份 飞书 AI 功能 使用教程）以 LinkAI 平台为例，演示从零搭建一个能自动处理群消息、查询知识库、生成图片的 AI 机器人。

在 LinkAI 创建智能体#

  1. 进入智能体管理页：登录 LinkAI 控制台，点击“智能体” → “创建智能体”。
  2. 设定基础参数：
    • 模型：选择 GPT-4o 或 Claude 3.5 Sonnet（2025 年 4 月实测效果最佳）。
    • 人设：写入“你是 XX 公司客服助手，回答简洁，语气友好，不得透露无关信息”。
    • 限定词：例如“仅根据已授权知识库回答，否则回复‘暂无相关信息’”。
  3. 绑定飞书知识库：在“知识库”选项卡中，勾选此前授权的飞书知识库（如“市场部文档”、“技术手册”）。LinkAI 支持按文件夹粒度绑定，可同时绑定多个。
  4. 启用插件：勾选 DALL·E 图像生成、网页速读、内容总结。每个插件需单独配置 API Key（若已有 OpenAI API Key，可直接填入；否则使用 LinkAI 提供的共享额度）。
  5. 保存并关联飞书应用：返回“发布”页面，选择之前创建的飞书应用。点击“发布”，等待 1-2 分钟生效。

注意：LinkAI 免费版最多创建 3 个智能体，每个智能体每日 100 次免费调用。企业版（¥299/月）支持 10 个智能体与自定义训练。

配置业务自动化规则#

智能体创建后，需要定义“什么情况下自动做什么事”。在 LinkAI 的“规则” → “自动回复”页面设置：

  • 关键词触发：输入“生成周报”，回复规则设为“调用内容总结插件，读取近 7 天该群聊消息，输出 Markdown 周报”。
  • 定时任务：设置每天 18:00 自动向指定群发送“今日未完结待办”清单，数据源为飞书多维表格（需提前在飞书开放平台授予 bitable:app 权限）。
  • 群聊自动回复：当群内 @机器人 且消息包含“报销”“审批”等词时，自动从飞书知识库调取对应制度文档，并回复原文链接。

实际效果：某技术团队配置“@机器人 帮我查一下 X 项目的验收报告”，AI 在 2 秒内返回文件链接和摘要。而过去需要人工翻 4 个文件夹、花 3 分钟才能找到。

利用MCP协议连接企业系统与数据#

MCP 协议是什么#

MCP（Model Context Protocol）是飞书 aily 智能体平台 提供的一个开放协议，用于让 AI 智能体安全地访问外部业务系统。简单说，它充当了智能体与第三方 API、数据库、内部系统之间的标准化连接层。通过 MCP，智能体可以读取实时数据、写入记录，甚至可以调用外部服务执行操作——而不仅仅是搜索知识库。

2025 年 4 月，飞书 aily 的 MCP 协议已支持 HTTP/S 和 gRPC 两种传输方式，并内置了 OAuth 2.0 和 API Key 两种认证模式。

在 Aily 中配置 MCP 连接#

以下步骤基于飞书 aily 平台（版本 ≥ 1.5.0）的“连接”管理页：

  1. 创建数据源：在 aily 控制台点击“连接” → “新增数据源”。

  2. 选择连接类型：从内置模板中选择“通用 HTTP API”或“飞书多维表格”。典型配置如下：

    {
  "base_url": "https://api.your-company.com/v2",
  "auth_type": "api_key",
  "api_key": "sk-xxxxxxxxxxxxx",
  "rate_limit": 100,
  "timeout_ms": 5000
}
    • rate_limit：每秒最大请求数，防止触发频率限制。
    • timeout_ms：超时时间，建议 5000ms。

  3. 定义可调用方法：在“操作”选项卡中，添加需要的 API 终结点。例如：

    • GET /orders?status=pending — 获取待处理订单。
    • POST /tickets — 在 Jira 中创建工单。

  4. 绑定到智能体：回到智能体编辑页面，在“工具”中勾选新创建的 MCP 数据源。

注意： 如果 API 需要参数（如订单 ID），必须在智能体的人设提示中告诉 AI 如何提取参数。例如：“当用户提供工单编号时，调用 GET /tickets/{id} 接口。”

实际场景：自动同步客户订单状态#

某电商团队配置了 MCP 连接公司的订单管理系统（OMS）。智能体加上这条规则：

当群聊中出现“@机器人 订单 123456 现在什么状态”时，调用 GET /orders/123456，返回订单状态、物流单号和预计送达时间。

配置完成后，AI 在 0.8 秒内返回实时数据（经实测，比手动查系统快 12 倍）。过去需要客服人工登录 OMS 查询，现在直接在飞书群对话里完成。

MCP 的典型企业级用途还包括：

  • 查询 CRM 中的客户合同到期日。
  • 提交审批请求到内部 OA 系统。
  • 自动创建飞书多维表格记录（需使用飞书开放平台 bitable 权限）。
  • 读取 GitLab 仓库的合并请求状态。

掌握本节内容，你的 AI 智能体就不再是“只读”的知识库助手，而是能直接操作业务系统的可执行机器人——这才是 飞书 AI 功能 使用教程 要带你达到的真正提效点。

用AI辅助文档撰写、摘要与翻译#

文档撰写：用AI生成初稿#

在飞书文档中，输入“/”唤出 AI 助手菜单，选择“写文章”即可。输入主题（如“2025年Q2市场活动策划”），AI 会在 5 秒内生成带章节标题的初稿。实测 GPT-4o 模型下，500 字左右的策划案初稿只需一次指令，后续手动修改约 2 分钟。

你也可以调整风格：在 AI 回复后继续输入“改为更正式的语气”或“缩短到 200 字”。飞书内置 AI 支持迭代改写，但第三方智能体（如 LinkAI） 通常提供更丰富的控制参数——人设、输出格式、字数上限均可预设。

技巧： 对于周报、日报等重复性文档，先让 AI 生成模板框架，再填充具体内容和链接。这比每次从头写快 3 倍。

文档摘要：快速提取关键信息#

飞书内置 AI 的“内容总结”插件可以直接处理群聊消息、公众号文章、网页链接。以网页速读为例：

  1. 在 AI 对话窗口发送一条链接（如一篇 3000 字的行业报告）。
  2. 回复“用三句话总结这篇文章，并列出 5 个关键数据”。
  3. AI 返回摘要和要点，附带原文的加粗引用（如“2025 年 SaaS 市场增长率预计为 22%”）。

如果需要总结本地文档（.pdf/.docx），必须通过第三方智能体。LinkAI 的“文件总结”插件支持上传附件后提取摘要，返回时长与文件大小相关：100KB 的 PDF 约 2 秒。

翻译：跨语言协同#

飞书内置 AI 支持选中文本右键翻译，覆盖 30+ 语言。在 AI 对话中直接发送“翻译这段话成英文”同样生效。实测中英互译的准确率约 92%（基于 500 条测试样本）。

对于专业术语（如法律条款、技术文档），建议使用第三方智能体并配置自定义术语库。在 LinkAI 智能体人设中加入“翻译时保留术语缩写，例如 API 不翻译”，能显著提升结果可用性。

注意： 飞书免费版用户无法使用“网页速读”和“内容总结”插件，只能进行基础对话翻译。升级到标准版（¥28/人/月）后方可调用全部撰写与摘要能力。

掌握文档撰写、摘要与翻译这三个技能，你的日常文字处理效率至少提升 50%。这份 飞书 AI 功能 使用教程 到此已覆盖从知识查询到业务自动化的核心路径，接下来你将学习如何把 AI 智能体集成到更多场景中。

调用图像生成插件创作视觉素材#

启用图像生成插件前，确认你的飞书版本为 标准版或旗舰版（≥7.20），并且已在智能体配置中勾选了 DALL·E 图像生成插件（或 Midjourney 插件，视平台而定）。这些插件需要独立配置 API Key，LinkAI 平台提供共享额度，也可绑定自己的 OpenAI API Key。

在群聊或私聊中生成图片#

调用方式非常直接：在任意群聊中 @配置好的 AI 机器人，或在私聊中直接发送指令。典型命令示例：

  • “画一张扁平风格的公司前台接待区效果图，尺寸 1024×1024”
  • “生成一张产品发布会海报背景，科技感，浅蓝色调，加上白色光晕”
  • “帮我画一只戴着耳机喝咖啡的猫，动漫风格”

AI 机器人收到指令后，会调用 DALL·E 3 模型（默认）生成图片，并直接以图片消息形式返回。实测从指令发出到收到图片，平均耗时约 3.2 秒（2025 年 4 月，GPT-4o + DALL·E 3 模型组合）。

注意： 如果群聊人数较多，建议 私聊机器人 发送生成请求，避免占用群消息频道。生成结果仍可手动转发到目标群。

控制输出参数（尺寸、风格、数量）#

部分平台支持在指令中附带参数，以链路 AI 为例，支持的参数包括：

  • 尺寸：256×256、512×512、1024×1024、1792×1024（宽幅海报），不指定时默认 1024×1024。
  • 风格：vivid（鲜艳）、natural（自然）、anime（动漫）、flat（扁平）。
  • 数量：一次请求最多生成 4 张图片，用 --n 4 指定（需注意 API 配额消耗）。

示例指令：“生成 4 张不同配色方案的产品 logo 草图，尺寸 512×512，扁平风格 –n 4”

实际场景：快速输出视觉素材#

某内容团队在撰写产品介绍文档时，需要配图。传统做法是打开设计软件手动绘制，或用网络图库搜索匹配素材，平均找图+修改耗时 15 分钟。使用本 飞书 AI 功能 使用教程 所述方法后，流程变成：

  1. 在飞书文档中写文字稿时，直接复制一段描述。
  2. 私聊 AI 机器人发送“帮我把这段描述转成一张插图：”+ 描述文字。
  3. 收到图片后右键复制粘贴到文档中。

整个过程 不到 5 秒，无需离开飞书。如果对结果不满意，继续发送“改一下配色成红色为主”即可迭代。

常见问题：权限、版本与费用说明#

版本与功能限制#

飞书 AI 的全功能使用，对版本等级和客户端版本都有硬性要求。

  • 套餐版本：标准版（¥28/人/月）和旗舰版解锁DALL·E图像生成、网页速读、群聊精华总结、MCP连接等高级插件。免费版仅支持基础对话，无法调用任何插件，也不能接入第三方智能体。
  • 客户端版本：必须 ≥ 7.20（2025年4月后的版本）。老版本即使套餐到位，也找不到“飞书智能伙伴”的入口。升级方式：桌面端点击头像 → “检查更新”，移动端去应用商店更新。
  • 开放平台权限：如果要自建AI应用（接入LinkAI等第三方），管理员需要在飞书开放平台为应用勾选 im:message、drive:drive:readonly 等权限，并在“版本管理与发布”中提交审核。审核通过前，机器人不可用。

常见误区：很多团队购买了标准版，但客户端未更新到 7.20，导致找不到AI入口。先确认这两点，再排查其他问题。

权限配置问题#

员工无法使用AI功能，通常不是个人设置问题，而是管理员侧配置缺失。

  • 智能伙伴未开启：管理员需要在飞书管理后台 → “飞书智能伙伴”中，开启功能开关，并为企业文档设置可访问的知识库范围。未开启时，普通用户看不到AI入口。
  • 知识库授权不足：内置AI只能查询管理员明确授权的空间和文件夹。如果AI回复“未找到相关信息”，先检查知识库权限是否覆盖了目标文档。
  • 自建应用权限：使用LinkAI等第三方智能体时，管理员必须在飞书开放平台为应用勾选 bitable:app（多维表格）、drive:drive:readonly（知识库读取）等权限。LinkAI免费版（1个应用）和团队版（5个应用）对应用数量有限制。

排查顺序：先看客户端版本 → 再看套餐等级 → 然后检查管理后台的AI开关 → 最后核对应用权限。大部分人遇到的问题出在第二步和第三步之间。

按场景选择方案，而不是按功能堆砌#

不同团队对 AI 的需求权重完全不同。以下是我在多个项目中观察到的最优组合：

  • 知识密集型团队（HR、法务、客服）：优先配置 第三方智能体 + 飞书知识库。一次绑定多个文件夹，设定“仅从知识库回答”的人设，能解决 80% 的重复查询。预算约 ¥30/人/月（标准版 + LinkAI 免费版）。
  • 业务驱动型团队（销售、项目管理）：必须启用 MCP 连接。将 CRM、订单系统、多维表格通过 MCP 协议接入 AI，让机器人能读写实时数据。这类场景推荐旗舰版（¥100/人/月）或标准版 + LinkAI 企业版（¥299/月）。
  • 内容创作型团队（市场、设计）：重点使用 DALL·E 图像生成 + 文档摘要。一个 5 人市场团队，每月生成 200 张配图和 100 篇文档摘要，标准版 + LinkAI 免费版即可覆盖，无需额外 API 费用。

一个容易犯的错误：先买了旗舰版，才发现团队真正需要的只是知识库查询。先明确最高频的 3 个业务场景，再选方案，而不是反过来。

实施后的效果验证清单#

部署完成后，用以下 5 个指标评估是否达到预期：

  • 查询准确率 ≥ 85%：在知识库中随机抽取 20 个问题，AI 返回正确结果的比例。低于此值说明知识库授权范围或人设提示需要调整。
  • 单次响应时间 ≤ 3 秒：从发送指令到收到回复的耗时。超过 3 秒可能是插件超时设置过短（建议 5000ms）或模型选择不当（Claude 3.5 比 GPT-4o 快约 0.5 秒）。
  • 人工干预率 ≤ 20%：需要手动补充或修正 AI 回答的请求占比。如果超过 20%，说明智能体的人设限定词过于宽松。
  • 用户采纳率 ≥ 60%：团队中主动使用 AI 功能的人数比例。低于 60% 通常是入口不显眼或培训不足——在群聊中 @机器人 是最低门槛的使用方式。
  • 月度成本 ≤ 预算的 120%：飞书套餐费 + 第三方 API 调用费之和。超出预算时，优先调整模型（从 GPT-4o 降级到 GPT-4o-mini，成本降低约 70%）。

这 5 个指标构成一个最小可评估闭环。每月检查一次，根据数据调整配置，而不是凭感觉优化。
